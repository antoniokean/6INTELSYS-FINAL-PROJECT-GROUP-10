# Speech Command Recognition + RL Threshold Tuning

This repository contains a speech command recognition pipeline using multi-channel log-Mel spectrogram features, a residual CNN classifier, and RL-based threshold tuning for cost-sensitive decision control.

## Environment Setup

Recommended Python version:

- `Python 3.11`

Install options:

1. `conda env create -f .\environment.yml` - create the Conda environment
2. `conda activate speech-command-rl` - activate the Conda environment
3. `python -m pip install -r .\requirements.txt` - install from the requirements file
4. `python -m pip install certifi kagglehub librosa matplotlib numpy torch` - install the core project packages directly

Optional package used in the Conda environment for notebooks:

`python -m pip install jupyter` - install Jupyter for notebooks

If `torch` needs a system-specific build for your CPU/GPU, install the correct PyTorch build first, then run `python -m pip install -r .\requirements.txt`.

To restore the dataset into the project, run `python .\get_data.py` from inside the [data folder](/C:/-Programming/Project-root/6INTELSYS-FINAL-PROJECT-GROUP-10/data).

## Quick Start

1. Check or restore the dataset:
   `cd .\data` then `python .\get_data.py`
2. Build the dataset manifest:
   `python .\src\data_pipeline.py`
3. Run the full pipeline with one command:
   `python .\run.py --preset medium_end --no-open-files`
4. Read saved results separately when needed:
   `python .\src\eval.py --preset medium_end`

For this project submission, `medium_end` is the official final preset.

## Training Presets

- `test`: fastest sanity-check run on a very small subset; uses `train=256`, `val=64`, `test=64`
- `low_end`: lightweight training setup for weaker CPU/RAM limits; uses `train=1000`, `val=200`, `test=200`
- `medium_end`: balanced default and final reporting preset; uses `train=10000`, `val=1000`, `test=1000`
- `high_end`: longest run that uses the full kept dataset when available for extra reference experiments

Each preset writes its own outputs under:

- `experiments/configs/<preset>/`
- `experiments/logs/<preset>/`
- `experiments/results/<preset>/`

Running the same preset again overwrites that preset's files, while other presets keep their own separate outputs.

## Model Setup

- Input data: 1-second speech clips loaded at 16 kHz from the dataset manifest generated by `src/data_pipeline.py`
- Audio features: normalized 3-channel features built from log-Mel, delta, and delta-delta spectrograms with `64` Mel bins, `n_fft=1024`, and `hop_length=256`
- CNN architecture: residual squeeze-excitation CNN with a convolutional stem, progressive channel expansion (`32 -> 64 -> 96 -> 128 -> 160`), adaptive pooling, and a small MLP classifier head
- Training objective: multi-class speech command classification with cross-entropy loss, label smoothing, and optional class weighting
- Optimizer: `AdamW` with a `ReduceLROnPlateau` learning-rate scheduler
- Data robustness: optional time shifting, additive/background-noise augmentation, SpecAugment-style masking, balanced sampling, and feature caching for faster reruns
- Evaluation metrics: validation/test loss, accuracy, Macro-F1, and RL threshold-tuning cost metrics
- RL threshold tuning: a Q-learning agent searches a confidence-threshold grid on validation predictions, then applies the tuned threshold to test predictions to reduce asymmetric expected cost

## Safe Ablation Runs

Use ablations only when you want a separate comparison run. Main preset runs still write to the normal preset folders, while ablations are isolated under that preset's own `ablations` folder.

- No augmentation:
  `python .\run.py --preset medium_end --ablation no_augment --no-open-files`
- No balancing:
  `python .\run.py --preset medium_end --ablation no_balance --no-open-files`
- Run both ablations:
  `python .\run.py --preset medium_end --ablation all --no-open-files`

When an ablation is selected, outputs are written under:

- `experiments/configs/<preset>/ablations/<ablation>/`
- `experiments/logs/<preset>/ablations/<ablation>/`
- `experiments/results/<preset>/ablations/<ablation>/`

This keeps the main preset result untouched in:

- `experiments/results/<preset>/`

## Baselines

- Simple non-DL baseline:
  `python .\src\non_dl_baseline.py --preset medium_end`
- Deep-learning baseline:
  `python .\run.py --preset medium_end --no-open-files`

The non-DL baseline uses the same manifest split and the same 3-channel log-Mel features as the CNN, but replaces the neural network with a nearest-centroid classifier. Its outputs are saved under:

- `experiments/configs/<preset>/baselines/nearest_centroid/`
- `experiments/results/<preset>/baselines/nearest_centroid/`

## Outputs

- `experiments/configs/<preset>/run_config.json`: saved run settings
- `experiments/logs/<preset>/history.json`: epoch-by-epoch training history
- `experiments/logs/<preset>/training_curves.png`: loss and accuracy curves
- `experiments/logs/<preset>/rl_threshold_history.json`: RL threshold-tuning episode history
- `experiments/results/<preset>/summary.json`: main metrics, including Macro-F1 and threshold-tuning summary
- `experiments/results/<preset>/threshold_tuning.json`: full threshold search and RL tuning details
- `experiments/results/<preset>/final_metrics.json`: focused final metrics for presentation
- `experiments/results/<preset>/final_metrics.md`: readable final-metrics summary
- `experiments/results/<preset>/evaluation_report.json`: compact machine-readable evaluation report
- `experiments/results/<preset>/evaluation_report.md`: report-ready text summary
- `experiments/results/<preset>/confusion_matrix.png`: confusion matrix PNG for the final evaluated split
- `experiments/results/<preset>/confusion_matrix_normalized.png`: row-normalized confusion matrix PNG
- `experiments/results/<preset>/confusion_matrix.json`: raw confusion-matrix counts and label order
- `experiments/results/<preset>/macro_f1_bar_chart.png`: presentation-ready validation vs test Macro-F1 graph
- `experiments/results/<preset>/sample_log_mel.png`: example log-Mel spectrogram grid from the run
- `experiments/results/<preset>/baseline_cnn.pt`: trained model weights
- `experiments/results/<preset>/baselines/nearest_centroid/summary.json`: simple non-DL baseline metrics
- `experiments/results/<preset>/baselines/nearest_centroid/confusion_matrix.png`: simple non-DL baseline confusion matrix
